Multi-Agent Conversation UX Design

Response Modes

Sequential: Agents respond in order, seeing previous responses. Parallel: All respond simultaneously.

Agent Requests

Agents can request input from other agents. User can approve/deny to maintain control.

Context Control

Users select which messages agents can see. Prevents context overload.

UX Behavior Recommendations

1 Response Modes

  • Sequential: Agents see previous responses, build on each other
  • Parallel: All agents respond independently with same context
  • Let users switch modes per message

2 Queue Management

  • Show "thinking" indicators with agent icons
  • Display queue order and estimated time
  • Allow users to skip/reorder queue

3 Agent Requests

  • Agents can request input from other agents
  • Show approval UI inline with message
  • Option to auto-approve trusted interactions

4 Context Control

  • Let users select which messages agents see
  • Show "context provided" summary with each message
  • Smart defaults: include relevant recent messages

5 Visual Threading

  • Use connection lines to show message flow
  • Color-code by agent for quick scanning
  • Show branch points for parallel conversations

6 User Control

  • Always let users interrupt/stop agents
  • Provide "skip this response" option
  • Allow editing agent responses before proceeding
